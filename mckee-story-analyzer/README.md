# McKee Story Analyzer

基于罗伯特·麦基《故事》《对白》《人物》三部曲的理论框架，对剧本、小说等文学创作进行专业点评与修改优化的 AI skill。像一个文学评论家那样工作——先做冷静的理论诊断，再给出可操作的修改方案。

## 它不是什么

- 不是泛泛的"写得不错/写得不好"——每个诊断都引用麦基理论中的具体概念
- 不是只会夸的作品鉴定师——会指出问题，并给出示范改写
- 不是拿长片标准衡量一切的教条主义者——内置格式自适应系统

## 五大分析维度

| 维度 | 核心关注 |
|------|---------|
| **故事结构** | 五部分结构、激励事件、鸿沟原理、高潮设计的必然性与意外性 |
| **人物设计** | 人物塑造 vs 真人物、欲望层次（自觉/不自觉）、人物弧光 |
| **对白与文本** | 对白的三功能、潜文本、展示 vs 讲述、叙述节奏 |
| **类型与体裁** | 类型约定与创新、核心价值的不可逆转变 |
| **场景设计** | 节拍分析、转折点、价值转变、场景与序列的关系 |

## 格式自适应系统

这是本 skill 区别于其他分析工具的核心设计。麦基的理论基于长片剧本（90-120分钟），直接套用在 2000 字的极短篇上会产生系统性误判。因此内置了三档标准：

| 篇幅级别 | 字数 | 分析尺度 |
|----------|------|---------|
| 长片/长篇 | >5万 | 完整麦基五部分结构标准 |
| 中篇 | 1-5万 | 简化结构，允许人物弧浓缩 |
| 极短篇/短片 | <1万 | 以"单一效果"为核心，放宽结构/人物维度要求 |

分析前强制判定篇幅级别并加载对应标准，避免用长片标准误判短篇。

## 工作流

```
Step 1: 接收与归类 → 判定格式、类型、篇幅级别
Step 2: 五维诊断 → 按需加载理论参考文件，逐维度点评优缺点
Step 3: 修改方案 → 针对每个问题给出可操作的修改建议 + 示范改写
Step 4: 综合总结 → 核心优势 + 问题优先级排序 + 总体建议
```

## 使用场景

- 上传小说/剧本片段，请求专业点评
- 找出作品的核心优缺点
- 基于麦基理论进行修改优化，获得示范改写
- 讨论故事创作的结构设计、人物塑造问题

## 安装

将 `mckee-story-analyzer.skill` 导入你的 AI 编码助手即可使用。
